## VГӯnculos relativos

Puedes definir enlaces relativos y rutas de imagen en los archivos representados para ayudar a que los lectores naveguen hasta otros archivos de tu repositorio.

Para crear una lista anidada mediante el editor web en GitHub o un editor de texto que usa una fuente monoespacial, como [Visual Studio Code](https://code.visualstudio.com/), puede alinear la lista visualmente. Escriba caracteres de espacio delante del elemento de lista anidado hasta que el carГЎcter de marcador de lista (<kbd>-</kbd> o <kbd>\*</kbd>) se encuentra directamente debajo del primer carГЎcter del texto del elemento encima de Г©l.

```markdown
1. First list item
   - First nested list item
     - Second nested list item
```

> \[!NOTE]
> En el editor basado en web, puede sangr o sangr una o varias lГӯneas de texto resaltando primero las lГӯneas deseadas y, a continuaciГіn, usando <kbd>tabulaciГіn</kbd> o <kbd>tabulaciГіn mayГәs</kbd>+ respectivamente.<kbd></kbd>

![Captura de pantalla de Markdown en Visual Studio Code que muestra la sangrГӯa de lГӯneas numeradas anidadas y viГұetas.](/assets/images/help/writing/nested-list-alignment.png)

![Captura de pantalla de GitHub Markdown representada que muestra un elemento numerado seguido de viГұetas anidadas en dos niveles diferentes de anidamiento.](/assets/images/help/writing/nested-list-example-1.png)

Para crear una lista anidada en el editor de comentarios en GitHub, que no usa una fuente monoespacial, puede examinar el elemento de lista inmediatamente encima de la lista anidada y contar el nГәmero de caracteres que aparecen antes del contenido del elemento. A continuaciГіn, escriba ese nГәmero de caracteres de espacio delante del elemento de lista anidado.

En este ejemplo, podrГӯa agregar un elemento de lista anidado bajo el elemento `100. First list item` de lista mediante la sangrГӯa del elemento de lista anidado como mГӯnimo de cinco espacios, ya que hay cinco caracteres (`100. `) antes `First list item`de .

```markdown

     - First nested list item
```

![Captura de pantalla de GitHub Markdown representada que muestra un elemento numerado precedido por el nГәmero 100 seguido de un elemento con viГұetas anidado de un nivel.](/assets/images/help/writing/nested-list-example-3.png)

Puede crear varios niveles de listas anidadas mediante el mismo mГ©todo. Por ejemplo, dado que el primer elemento de lista anidado tiene siete caracteres (`вҗЈвҗЈвҗЈвҗЈвҗЈ-вҗЈ`) antes del contenido `First nested list item`de la lista anidada, tendrГӯa que sangr el segundo elemento de lista anidado por al menos dos caracteres mГЎs (nueve espacios como mГӯnimo).

Para obtener mГЎs ejemplos, consulte el [GitHub EspecificaciГіn de Markdown con sabor](https://github.github.com/gfm/#example-265).
